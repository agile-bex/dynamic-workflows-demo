## What the card asked

A PySide6 desktop CRUD UI over the existing `todo_store` module ŌĆö add, list,
toggle complete/un-complete, delete ŌĆö launchable by the human with `uv run app`.
Hard constraints: all data goes through `todo_store` (no new persistence), an
`app` script wired into `pyproject.toml`, **no tests**, and **do not run the GUI**
(only `python -c "import ..."` to catch import/syntax errors).

## Homework against the real code

- Dependency satisfied: `todo-store` is in `tasks/done/`, and
  `src/todo_app/todo_store.py` exists with the full CRUD API I built against:
  - `create(title, done=False) -> dict`
  - `list() -> list[dict]` (returns copies; each todo has `id`, `title`, `done`)
  - `update(id, *, title=None, done=None) -> dict | None`
  - `delete(id) -> bool`
  The store owns its JSON file and tolerates missing/corrupt files. No UI logic
  in it. No mismatch ŌĆö proceeded.
- PySide6 was **not** installed (`import PySide6` -> ModuleNotFoundError). Since
  the app requires it and the card's "done" bar is "modules import cleanly", I
  added `PySide6>=6.6` to `[project.dependencies]` and ran `uv sync`
  (installed pyside6 6.11.1). This is required scope, not gold-plating.

## What I built

- `src/todo_app/app.py` ŌĆö `TodoWindow(QMainWindow)` + `main()`:
  - Add row (line edit + Add button, also Enter to add).
  - `QListWidget` with a user-checkable item per todo; the checkbox toggles
    `done` via `store.update(id, done=...)`.
  - "Delete selected" button calls `store.delete(id)` (info prompt if nothing
    selected).
  - `_refresh()` rebuilds the list from `store.list()` (signals blocked during
    rebuild so programmatic check-state changes don't re-fire `itemChanged`).
  - Each item carries its todo id in `Qt.ItemDataRole.UserRole`.
  - No persistence logic here ŌĆö every mutation goes through `TodoStore`. The
    store file path is `todos.json` in the cwd (the store owns the file).
- `pyproject.toml` ŌĆö added `app = "todo_app.app:main"` under `[project.scripts]`
  and `PySide6>=6.6` to dependencies.
- `.gitignore` ŌĆö added `todos.json` (runtime data the app writes), so a human
  running `uv run app` won't dirty the tree.
